## US-RGPD-01 - Exporter ses donnees personnelles (SAR Art. 15 / portabilite Art. 20)

Epic : EPIC RGPD — Droits utilisateur
Backlog initial : US-18 (Could → Must P4)
Priorite MoSCoW : MUST
Sprint cible : Sprint 5 (TASK-27)
Points : 3

User story :
En tant qu'utilisateur authentifie,
je veux recuperer l'integralite de mes donnees personnelles au format JSON et CSV,
afin d'exercer mon droit d'acces (Art. 15) et de portabilite (Art. 20).

Analyse INVEST :
  I — Endpoint dedie, independant des autres features.
  N — Premiere version : JSON + CSV ; ZIP en bonus (COULD).
  V — Repond directement a la demande Hugo PETIT.
  E — Modeles User, Quiz, Question deja identifies.
  S — Limite a l'utilisateur connecte (request.user).
  T — Tests anti-fuite inter-comptes possibles.

Criteres d'acceptation (CA-J3B-1, CA-J3B-2, CA-J3B-3, CA-J3B-4) :

  CA-01  GET /api/accounts/me/export/ retourne HTTP 200 pour un user authentifie.
  CA-02  Endpoint protege : 401/403 si non authentifie.
  CA-03  Filtrage strict request.user sur TOUTES les tables (anti-leak).
         Test : user A exporte → zero donnee de user B (quiz, reponses, logs).
  CA-04  Export JSON structure et valide (Content-Type application/json).
  CA-05  Export CSV ou second format tabulaire equivalent (query param ?format=csv
         ou header Accept).
  CA-06  Categories obligatoires dans l'export (6) :
         (1) donnees de compte (email, dates inscription/connexion) ;
         (2) cours / textes uploades (source_text des quiz) ;
         (3) quiz generes (titre, date, questions) ;
         (4) reponses utilisateur et scores ;
         (5) signalements emis (QuestionReport si modele present ; sinon cle vide
             documentee) ;
         (6) logs / demandes RGPD lies a l'utilisateur (DataRequest).
  CA-07  Aucun mot de passe, token, hash ou secret technique dans l'export.
  CA-08  Header Content-Disposition: attachment ; nom de fichier horodate
         edututor_export_<YYYY-MM-DD>.json (ou .csv).
  CA-09  Bouton « Exporter mes donnees » visible sur /profile (utilisateur connecte).
  CA-10  Telechargement declenche sans envoi asynchrone par email.
  CA-11  Une entree DataRequest (type=export, statut=completed) est creee
         automatiquement (cf. US-RGPD-03).
  CA-12  Durees de retention documentees dans l'export ou pages legales
         (reference p3bis-politique-retention.txt : PDF 30 j, quiz 12 mois).

DoR :
  Lister les modeles Django exportables (User, Profile, Quiz, Question).
  Valider le format JSON (schema) et CSV (colonnes minimales).
  Definir le comportement si signalements absents (J4 non livre).

DoD :
  Endpoint fonctionnel + tests anti-fuite.
  Bouton frontend operationnel.
  DataRequest cree a chaque export.
  Demo PO : export du compte hugo.petit@test.local reproductible.

## US-RGPD-02 - Supprimer ses donnees personnelles (effacement Art. 17)

Epic : EPIC RGPD — Droits utilisateur
Backlog initial : US-05 (Should → Must P4)
Priorite MoSCoW : MUST
Sprint cible : Sprint 5 (TASK-28)
Points : 2

User story :
En tant qu'utilisateur authentifie,
je veux supprimer definitivement mon compte et toutes mes donnees personnelles,
afin d'exercer mon droit a l'effacement (Art. 17).

Analyse INVEST :
  I — Action compte separee de l'export.
  N — Suppression immediate en base ; anonymisation partielle si contrainte legale.
  V — Droit fondamental ; demande aussi par Hugo PETIT.
  E — Tables identifiees : User, Quiz, Question, Token.
  S — Bouton deja present dans le kit.
  T — Verification post-suppression en base.

Criteres d'acceptation :

  CA-01  Bouton « Supprimer mon compte » accessible depuis /profile.
  CA-02  Confirmation explicite requise (mot de passe ou dialogue de confirmation).
  CA-03  Suppression en cascade : compte + quiz + questions + scores + reponses.
  CA-04  Token d'authentification invalide apres suppression.
  CA-05  Reconnexion impossible avec les memes identifiants.
  CA-06  Donnees plus accessibles via l'API (GET historique → 404 ou liste vide).
  CA-07  Donnees personnelles retirees des vues enseignantes (US-J1) si presentes.
  CA-08  Trace minimale conservee dans DataRequest (type=delete, sans recopie
         des donnees supprimees) — cf. politique retention section 3.6.
  CA-09  Email de confirmation de suppression envoye (best-effort Brevo).
  CA-10  Delai de suppression effective documente : immediate en base ;
         alignement Art. 17 al. 3 (30 jours max si backup — mention pages legales).

DoR :
  Cartographier les FK User → Quiz → Question.
  Valider avec PO : conservation audit trail SAR seul (3 ans, donnees minimales).

DoD :
  TASK-28 execute : zero orphelin en base apres suppression test.
  DataRequest cree a chaque suppression.
  Test manuel : compte supprime → export impossible.

## US-RGPD-03 - Tracer les demandes RGPD (audit trail SAR)

Epic : EPIC RGPD — Audit trail SAR
Priorite MoSCoW : MUST
Sprint cible : Sprint 5
Points : 2

User story :
En tant qu'administrateur / responsable de traitement,
je veux conserver une trace minimale des demandes RGPD (export, suppression),
afin de prouver le traitement d'une SAR conformement aux attentes CNIL (CA-J3B-6).

Analyse INVEST :
  I — Modele DataRequest autonome.
  N — V1 : champs minimaux ; pas de back-office admin complet.
  V — Preuve de conformite pour Hugo PETIT et futurs SAR.
  E — 1 modele + 2 hooks (export, delete).
  S — Pas d'interface admin obligatoire pour MVP.
  T — Assertion ORM : 1 export → 1 DataRequest.

Criteres d'acceptation (CA-J3B-6) :

  CA-01  Modele DataRequest persiste en base (migration Django).
  CA-02  Champs obligatoires : user (FK), request_type (export | delete),
         status (received | in_progress | completed | failed),
         created_at, completed_at (nullable), export_file_hash (nullable).
  CA-03  Chaque export (US-RGPD-01) cree une entree status=completed.
  CA-04  Chaque suppression (US-RGPD-02) cree une entree status=completed.
  CA-05  L'audit trail NE contient PAS le contenu integral des donnees exportees.
  CA-06  Conservation 3 ans (politique retention section 3.6).
  CA-07  hash du fichier exporte stocke pour integrite (SHA-256 du JSON genere).
  CA-08  Accessible en lecture admin Django (optionnel MVP) ou via shell.

DoR :
  Schema DataRequest valide par l'equipe.
  Statuts et types enumeres documentes.

DoD :
  Migration appliquee.
  Creation automatique verifiee par test unitaire.
  Mention dans politique retention section 3.6 coherente.

afin de comprendre comment EduTutor IA traite mes donnees personnelles (Art. 13).

  I — Pages statiques React, independantes du backend.
  N — Contenu minimal acceptable ; detail juridique dans p3bis-politique-retention.txt.
  V — Transparence ; prerequis CA-J3B-5.
  E — 4 pages deja squelettes dans le kit.
  S — Remplacement des placeholders vides.
  T — Navigation footer + contenu non vide.
