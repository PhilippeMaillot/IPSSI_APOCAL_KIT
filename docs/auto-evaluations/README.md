# 🪞 Auto-évaluation individuelle (obligatoire)

> ⚠️ **Chaque étudiant·e** dépose **ici** (`docs/auto-evaluations/`) sa **propre**
> auto-évaluation, à **deux échéances**. C'est **individuel** et **obligatoire** :
> un oubli est **sanctionné** (voir [§ Malus](#️-malus-en-cas-doubli)).

## En 30 secondes

1. **Copiez** le modèle [`auto-evaluation-individuelle.xlsx`](./auto-evaluation-individuelle.xlsx).
2. **Renommez** votre copie selon la convention : `NOM_Prenom_PROMO.xlsx`
   (ex. `DUPONT_Marie_M1.xlsx`).
3. **Remplissez le bon onglet** à chaque échéance, puis **commitez** votre fichier
   dans ce dossier.

## Le classeur — 3 onglets

| Onglet | Quand | Ce que vous remplissez |
|---|---|---|
| ① **Critères & sens** | *(référence)* | Rappel des **14 critères** des deux notes /20 |
| ② **Mi-parcours (mer. 17h)** | **avant mercredi 17h00** | Vous vous auto-notez sur les **7 critères techniques** (bilan de votre semaine) |
| ③ **Oral (jeu. 17h)** | **avant jeudi 17h00** | Vous vous auto-notez sur les **7 critères de l'oral**, pendant vos **répétitions** de soutenance |

> 🧭 L'auto-évaluation est **formative** : les notes que vous vous donnez **ne sont
> pas évaluées**. Ce qui est contrôlé, c'est **le fait de la réaliser et de la
> commiter à temps**.

## ⚖️ Malus en cas d'oubli

Chaque échéance manquée — **non commitée**, **mauvais nommage**, ou **déposée hors
délai** — retire :

- **−1 point** à votre note **« Travail technique & gestion du projet »**, **et**
- **−1 point** à votre note **« Soutenance orale »**.

C'est **cumulatif** : oublier les **deux** échéances = **−2 / −2**. Le malus est
appliqué **individuellement** par le jury, qui vérifie ce dossier.

## Convention de nommage (à respecter STRICTEMENT)

```
docs/auto-evaluations/NOM_Prenom_PROMO.xlsx
```

- `NOM` en **MAJUSCULES**, `Prenom` en Capitale, `PROMO` = `B3` ou `M1`.
- Exemple : `docs/auto-evaluations/DUPONT_Marie_M1.xlsx`.
- **Un seul fichier par étudiant** : vous le re-commitez **mercredi** (après l'onglet ②)
  puis **jeudi** (après l'onglet ③). Pas de date dans le nom : c'est la date du commit
  qui fait foi.

## Pourquoi ?

S'auto-évaluer **à froid** sur les critères réels, c'est repérer ses angles morts
avant qu'il ne soit trop tard — exactement la posture agile attendue (*inspect &
adapt*). Soyez **honnête** : l'objectif est de vous **améliorer**, pas de vous noter.
