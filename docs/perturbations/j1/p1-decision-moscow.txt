1. CONTEXTE DE LA PERTURBATION
================================
Le Product Owner annonce a 14h00 l'ajout d'une cible secondaire confirmee :
Mme Sophie Lefevre, 42 ans, professeure de Communication BTS Lyon, 28 etudiants.

Ses besoins declares :
  - Suivre les scores de ses 28 etudiants apres chaque quiz
  - Reperer rapidement les etudiants qui decrochent
  - Transmettre des conseils personnalises aux etudiants en difficulte

Cette persona etait deja identifiee dans notre cadrage matinal comme
"cible secondaire" (artefact-2, artefact-3). La perturbation confirme
son importance et exige une repriorisation explicite du backlog.

3. VERDICT POUR RELEASE 1 (mercredi 29/06 17h45 STRICT)
=========================================================

Mme Lefevre = CO-PRIMAIRE comme persona / SHOULD HAVE pour ses features specifiques

Note de coherence : la designation "Co-PRIMAIRE" signifie que Mme Lefevre est
une cible aussi importante que l'etudiant(e) du point de vue produit. Cela ne
signifie pas que toutes ses features sont Must Have. Les features Must Have
(F1-F6) servent deja les deux profils simultanement. Ce sont ses features
specifiques (dashboard classe, scores etudiants, reperage decrocheurs) qui sont
Should Have car elles ne bloquent pas le MVP etudiant et necessitent un modele
de donnees supplementaire (rattachement etudiant / classe / enseignant).

Justification argumentee :

a) Les features Must Have (F1 a F6) sont identiques pour etudiant et enseignant.
   Mme Lefevre peut utiliser EduTutor des la Release 1 sans developpement
   supplementaire : elle uploade son cours, obtient un quiz, le fait passer
   a ses etudiants. Le parcours exist deja dans le MVP.

b) Les features SHOULD (US-J1-01, US-J1-02) reutilisent les scores et
   l'historique deja prevus dans le MVP. Elles ne cassent pas le backlog
   initial et apportent une valeur directe a Mme Lefevre en Release 1
   si le temps le permet apres les MUST.

c) Les features COULD (US-J1-03, US-J1-04) necessitent un systeme de
   messagerie ou de rattachement classe qui depasse le perimetre Sprint 1-5.
   Elles sont ciblees Release 2 (jeudi 17h45).
