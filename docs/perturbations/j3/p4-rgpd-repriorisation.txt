1. CONTEXTE DE LA PERTURBATION
================================================================================

Le PO signale que la Release 1 doit etre conforme RGPD avant demonstration.
Deux obligations legales sont exigees :

  1. Droit a la portabilite (Art. 20 RGPD) :
     L'utilisateur doit pouvoir recuperer ses donnees personnelles
     sous format lisible (JSON ou CSV).

  2. Droit a l'effacement (Art. 17 RGPD) :
     L'utilisateur doit pouvoir supprimer son compte et toutes ses donnees.

  3. Obligation d'information (Art. 13 RGPD) :
     Des pages legales doivent etre accessibles depuis le site :
     - Mentions legales
     - Politique de confidentialite
     - CGU (Conditions Generales d'Utilisation)
     - Politique de cookies

Ces obligations existaient deja dans le cadrage initial mais leur statut
MoSCoW (Should / Could) ne les rendait pas prioritaires. La perturbation P4
les passe en MUST HAVE pour la Release 1.

3. MODIFICATIONS PRODUCT BACKLOG (artefact-6)
================================================================================

US-05 - Supprimer son compte (RGPD Art. 17)
  MoSCoW avant : Should Have
  MoSCoW apres : Must Have
  Sprint cible  : Sprint 5 (mercredi J3 14h-18h)
  Criteres d'acceptation mis a jour :
    1. Bouton "Supprimer mon compte" accessible depuis /profile
    2. Confirmation par saisie manuelle du mot de passe
    3. Suppression complete : compte + quiz + reponses + logs utilisateur
    4. Deconnexion immediate apres suppression
    5. Email de confirmation envoi a l'adresse supprimee
    6. Impossible de se reconnecter avec les memes identifiants apres suppression

US-18 - Exporter ses donnees RGPD (Art. 20)
  MoSCoW avant : Could Have
  MoSCoW apres : Must Have
  Sprint cible  : Sprint 5 (mercredi J3 14h-18h)
  Criteres d'acceptation mis a jour :
    1. Bouton "Telecharger mes donnees" accessible depuis /profile
    2. Export JSON contenant : profil utilisateur + historique quiz + reponses
    3. Fichier nomme : edututor-export-{user_id}-{date}.json
    4. Format lisible par machine et humain (JSON indenté)
    5. Toutes les donnees personnelles incluses (aucune omission)
    6. Telechargement declenche cote backend (pas de donnees en localStorage)
    7. Confirmation toast apres declenchement du telechargement
